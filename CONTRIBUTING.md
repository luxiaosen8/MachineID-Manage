# 贡献指南

感谢您对 MachineID-Manage 项目的兴趣！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题
- 使用 GitHub Issues 报告 Bug
- 提供详细的问题描述和复现步骤
- 包含系统环境信息（Windows 版本、Rust 版本等）
- 如果可能，提供错误日志或截图

### 功能建议
- 在 GitHub Discussions 中讨论新功能
- 描述用例和预期行为
- 解释为什么该功能对项目有价值

### 代码贡献

#### 准备工作
1. Fork 本仓库
2. 克隆您的 Fork 到本地
3. 设置开发环境

```bash
# 安装 Rust（如果尚未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 安装 Tauri CLI
cargo install tauri-cli

# 克隆项目
git clone https://github.com/YOUR_USERNAME/MachineID-Manage.git
cd MachineID-Manage
```

#### 开发流程
1. 创建功能分支: `git checkout -b feature/your-feature`
2. 遵循项目规范（见 AGENTS.md）
3. 编写测试并确保通过
4. 提交更改，遵循约定式提交规范
5. 推送到您的 Fork
6. 创建 Pull Request

#### 代码规范
- 遵循 AGENTS.md 中的开发规范
- 保持代码简洁、可读
- 添加必要的注释（中文）
- 更新相关文档

### 文档贡献
- 改进现有文档
- 翻译文档到其他语言
- 添加使用示例
