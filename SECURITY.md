# 安全政策

## 报告安全漏洞

我们非常重视项目安全。如果您发现任何安全漏洞，请遵循以下步骤：

### 报告流程
1. **不要**在公开的 GitHub Issues 中报告安全问题
2. 发送邮件至: [安全邮箱待添加]
3. 等待我们的回复（通常在 48 小时内）

### 报告内容
请包含以下信息：
- 漏洞描述
- 影响范围
- 复现步骤
- 可能的修复建议
- 您的联系方式（可选）

## 安全功能

### 注册表操作安全措施
- **备份验证**: 修改前自动备份，提供回滚功能
- **用户确认**: 所有危险操作需要用户明确确认
- **操作日志**: 记录所有注册表修改操作
- **输入验证**: 验证所有用户输入

### 代码安全实践
- **依赖审核**: 定期检查依赖的安全性
- **代码审查**: 所有代码变更需要审查
- **静态分析**: 使用 Clippy 和其他工具进行代码分析
- **测试覆盖**: 保持高测试覆盖率

## 安全最佳实践

### 用户建议
1. 始终从官方来源下载本软件
2. 验证下载文件的完整性
3. 使用前创建系统备份
4. 仅授予必要的权限
5. 定期更新到最新版本

### 开发安全
1. 遵循 Rust 安全最佳实践
2. 避免使用不安全的代码块
3. 正确处理错误和异常
4. 不在代码中存储敏感信息

## 依赖安全

我们使用以下工具监控依赖安全：
- `cargo audit` - 检查已知漏洞
- `cargo deny` - 依赖许可和安全检查
- Dependabot（GitHub）- 自动依赖更新

## 版本支持

### 当前支持版本
- Tauri 2.x
- Rust 1.70+

### 安全更新
我们承诺：
- 在发现安全漏洞后 7 天内发布修复
- 为最新版本提供安全更新
- 及时通知用户已知漏洞

## 合规性

- 本项目遵循负责任的披露流程
- 我们不会对安全研究人员采取法律行动
- 我们感谢帮助我们提高安全性的贡献者
